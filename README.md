
# 群聊管理插件 (Group Manager Plugin)

为 KiraAI 提供群聊管理功能，支持禁言、踢人、设置名片等操作。

## 功能特性

### 核心功能（默认启用）
- ✅ **禁言/解除禁言** - 对指定成员进行禁言管理
- ✅ **设置群名片** - 修改成员群名片
- ✅ **撤回消息** - 撤回群内的消息
- ✅ **查询成员列表** - 查看群成员简要信息
- ✅ **查询成员信息** - 查看指定成员的详细信息

### 可选功能（需手动开启）
- ⚠️ **踢出成员** - 将成员踢出群聊
- ⚠️ **全员禁言** - 开启/关闭全员禁言

## 安装

1. 将插件文件夹复制到 `data/plugins/`
2. 重启 KiraAI 或重新加载插件
3. 在 WebUI 中配置管理员（或手动编辑配置文件）

## 配置说明

配置文件位置：`data/config/plugins/group_manager.json`

```json
{
    "admin_qq_list": ["123456789", "987654321"],
    "allow_ai_autonomous": true,
    "enable_kick_user": false,
    "enable_whole_ban": false,
    "auto_check_admin": true,
    "log_operations": true
}
```

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `admin_qq_list` | 数组 | [] | **管理员QQ号列表**，只有这些用户可以使用群管功能（当 `allow_ai_autonomous` 为 `false` 时生效） |
| `allow_ai_autonomous` | 布尔 | true | **允许 AI 自主执行**：开启后，AI 自行决定发起的群管操作（如自动禁言违规用户）将跳过调用者身份检查，直接执行。关闭时，仅允许 `admin_qq_list` 中的用户命令执行 |
| `enable_kick_user` | 布尔 | false | 是否启用踢人功能（高危） |
| `enable_whole_ban` | 布尔 | false | 是否启用全员禁言功能（高危） |
| `auto_check_admin` | 布尔 | true | 是否自动检查Bot是否为群管理员 |
| `log_operations` | 布尔 | true | 是否记录群管操作日志 |

## 使用方法

### 1. 设置管理员

在 WebUI 的插件配置页面，添加管理员QQ号或手动编辑配置文件：
```json
{
    "admin_qq_list": ["你的QQ号", "朋友的QQ号"]
}
```

### 2. 使用群管功能

管理员在群内@Bot并发出指令，例如：
```
@Bot 把发广告的禁言10分钟
@Bot 查看群成员列表
@Bot 设置我的群名片为"大佬"
```

### 3. 开启可选功能

如需使用踢人或全员禁言，修改配置：
```json
{
    "enable_kick_user": true,
    "enable_whole_ban": true
}
```

### 4. 配置 AI 自主执行

默认情况下，`allow_ai_autonomous` 为 `true`，AI 可以根据对话内容自行决定执行禁言等操作（例如检测到刷屏、辱骂时自动处理）。如果你希望只有管理员通过命令才能触发群管功能，可将该开关关闭。

#### 提示词配置建议（重要）

当开启 `allow_ai_autonomous` 后，建议在 KiraAI 的人设提示词或群聊指令中根据需要加入以下规则，以确保 AI 正确区分“用户命令”与“自主判断”，避免被普通用户利用：

```markdown
## 群管操作安全规则
- 当用户**直接命令**你执行禁言、踢人等操作时（例如“把某人禁言”），你**必须**先确认该用户是否在你的管理员列表中。如果不在，应礼貌拒绝并说明“你没有权限”。
- 只有当你**自主判断**用户行为违规（如刷屏、辱骂、广告）时，你才可以**主动**调用群管工具，无需等待用户命令。
- 执行任何群管操作前，请简要说明原因（例如“检测到刷屏，禁言3分钟”）。
- 如果操作失败（如Bot不是群管理员），请提示用户联系群主设置Bot为管理员。
```

将上述提示词添加到 KiraAI 的角色设定中，可增强安全性，避免普通用户通过诱导 AI 执行违规操作。但并非绝对安全，请自行斟酌风险来是否开启一些高危选项。

## 权限说明

### 管理员验证
- 当 `allow_ai_autonomous` 为 `true` 时：
  - AI 自主发起的操作（非用户命令）直接放行
  - 用户命令仍会检查调用者是否在 `admin_qq_list` 中
- 当 `allow_ai_autonomous` 为 `false` 时：
  - 只有 `admin_qq_list` 中的QQ号可以使用群管功能
  - 机器人自身（系统消息、定时任务）通过 `allow_bot_self` 单独控制
- 非管理员调用工具时会返回：`❌ 用户不是插件的管理员`

### 群管权限
- Bot需要是**群管理员**才能执行群管操作
- 如果Bot不是群管理员，操作会失败

## 安全设计

1. **分层权限控制** - 区分用户命令与 AI 自主执行，支持灵活开关
2. **高危功能开关** - 踢人和全员禁言默认关闭，需手动开启
3. **操作审计日志** - 所有群管操作记录到日志，便于追溯
4. **错误提示明确** - 权限不足时明确告知原因
5. **提示词安全建议** - 提供示例规则，防止 AI 被滥用

## 故障排查

### 操作失败，提示"用户不是插件的管理员"
- 如果是由用户命令触发的：检查你的QQ号是否已添加到 `admin_qq_list`
- 如果是 AI 自主发起的：检查 `allow_ai_autonomous` 是否为 `true`
- 重新加载插件使配置生效

### 操作失败，提示"Bot不是群管理员"
- 在群设置中将Bot设为管理员

### 踢人/全员禁言功能不可用
- 检查配置中是否开启了 `enable_kick_user` / `enable_whole_ban`

### 获取适配器失败
- 检查Bot是否正常连接到QQ
- 查看日志中的适配器名称是否正确
