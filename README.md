# 群聊管理插件 (Group Manager Plugin)

为 KiraAI 提供群聊管理功能，支持禁言、踢人、设置名片等操作。

## 功能特性

### 核心功能（默认启用）
- ✅ **禁言/解除禁言** - 对指定成员进行禁言管理
- ✅ **设置群名片** - 修改成员群名片
- ✅ **撤回消息** - 撤回群内的消息
- ✅ **查询成员列表** - 查看群成员简要信息
- ✅ **查询成员信息** - 查看指定成员的详细信息

### 可选功能（需手动开启）
- ⚠️ **踢出成员** - 将成员踢出群聊
- ⚠️ **全员禁言** - 开启/关闭全员禁言

## 安装

1. 将插件文件夹复制到 `data/plugins/group_manager/`
2. 重启 KiraAI 或重新加载插件
3. 在 WebUI 中配置管理员（或手动编辑配置文件）

## 配置说明

配置文件位置：`data/config/plugins/group_manager.json`

```json
{
    "admin_qq_list": ["123456789", "987654321"],
    "enable_kick_user": false,
    "enable_whole_ban": false,
    "auto_check_admin": true,
    "log_operations": true
}
```

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `admin_qq_list` | 数组 | [] | **管理员QQ号列表**，只有这些用户可以使用群管功能 |
| `enable_kick_user` | 布尔 | false | 是否启用踢人功能（高危） |
| `enable_whole_ban` | 布尔 | false | 是否启用全员禁言功能（高危） |
| `auto_check_admin` | 布尔 | true | 是否自动检查Bot是否为群管理员 |
| `log_operations` | 布尔 | true | 是否记录群管操作日志 |

## 使用方法

### 1. 设置管理员

在 WebUI 的插件配置页面，添加管理员QQ号：

![配置示例](https://docs.kira-ai.top/images/group_manager_config.png)

或手动编辑配置文件：
```json
{
    "admin_qq_list": ["你的QQ号", "朋友的QQ号"]
}
```

### 2. 使用群管功能

管理员在群内@Bot并发出指令，例如：

```
@Bot 把发广告的禁言10分钟
@Bot 查看群成员列表
@Bot 设置我的群名片为"大佬"
```

### 3. 开启可选功能

如需使用踢人或全员禁言，修改配置：
```json
{
    "enable_kick_user": true,
    "enable_whole_ban": true
}
```

## 权限说明

### 管理员验证
- 只有 `admin_qq_list` 中的QQ号可以使用群管功能
- 非管理员调用工具时会返回：`❌ 用户不是插件的管理员`
- Bot自己（self_id）自动拥有所有权限

### 群管权限
- Bot需要是**群管理员**才能执行群管操作
- 如果Bot不是群管理员，操作会失败

## 工具列表

| 工具名 | 说明 | 权限要求 |
|--------|------|----------|
| `group_ban_user` | 禁言成员 | 插件管理员 |
| `group_unban_user` | 解除禁言 | 插件管理员 |
| `group_set_card` | 设置群名片 | 插件管理员 |
| `group_delete_msg` | 撤回消息 | 插件管理员 |
| `group_get_member_list` | 获取成员列表 | 插件管理员 |
| `group_get_member_info` | 获取成员信息 | 插件管理员 |
| `group_kick_user` | 踢出成员 | 插件管理员 + 配置开启 |
| `group_whole_ban` | 全员禁言 | 插件管理员 + 配置开启 |

## 安全设计

1. **严格的权限验证** - 每次工具调用都验证调用者身份
2. **高危功能开关** - 踢人和全员禁言默认关闭，需手动开启
3. **操作审计日志** - 所有群管操作记录到日志，便于追溯
4. **错误提示明确** - 权限不足时明确告知用户"用户不是插件的管理员"

## 故障排查

### 操作失败，提示"用户不是插件的管理员"
- 检查你的QQ号是否已添加到 `admin_qq_list`
- 重新加载插件使配置生效

### 操作失败，提示"Bot不是群管理员"
- 在群设置中将Bot设为管理员

### 踢人/全员禁言功能不可用
- 检查配置中是否开启了 `enable_kick_user` / `enable_whole_ban`

### 获取适配器失败
- 检查Bot是否正常连接到QQ
- 查看日志中的适配器名称是否正确
